# Act Two Catering — UX/UI Audit Report

> **Date:** 2026-03-18
> **Auditor:** Senior UX/UI + Code Quality Analysis
> **Version audited:** app.js (2,765 lines, 119KB), index.html, manifest.json, sw.js
> **Surfaces reviewed:** Marketing site, quote form, budget estimator, PWA shell
> **Method:** Full source-level analysis — every component, handler, style, and data object

---

## What This App Actually Is

Act Two Catering is a **marketing and inquiry PWA** for a premium catering company in South Jersey & Philadelphia. Users can browse 6 service types, view 10 menu items, explore 6 past event case studies, read 8 reviews, calculate pricing estimates, and submit a quote inquiry form.

**What it's NOT:** It does not book events, process payments, store user data, or send emails. The quote form shows a success message but the data goes nowhere.

## Main Conversion Goal

**Get visitors from browsing → submitted quote inquiry as fast as possible.**

The funnel is:
```
Land on site → Browse services/menu/events → Click "Book a Tasting" → Fill quote form → Submit
```

**The funnel is broken.** Form submissions disappear. The business never receives the lead.

## Architecture Assessment

### What's Good
- Hand-written, not generated. Consistent patterns throughout.
- Proper React hooks usage (useState, useEffect, useRef)
- Clean data layer: 6 const objects with consistent schemas
- Semantic HTML with ARIA attributes on interactive elements
- CSS-in-JS via template string — cohesive design system
- PWA shell with proper manifest, icons, and SW caching
- Responsive at 4 breakpoints (960, 768, 600, 480px)
- Touch targets meet 44×44px minimum
- prefers-reduced-motion respected

### What's Wrong
- **One file.** 2,765 lines, 119KB. Data, styles, components, routing, and app shell all in app.js.
- **No module system.** React loaded from CDN. No imports, no tree-shaking.
- **No build step.** Can't minify, split, or optimize.
- **No tests.** Zero test files, zero test patterns.
- **No error handling.** Zero try/catch blocks. Zero error boundaries.
- **No backend.** Form is a dead end.

---

## Visual Design

### Strengths
- Cohesive wine/gold/cream palette with proper CSS variables
- Playfair Display headings + DM Sans body — professional pairing
- Consistent 8px spacing grid (8, 16, 24, 32, 48, 64)
- 4 shadow levels defined and used consistently
- Card, pill, and badge patterns reused across all page types
- Hover and active states on all interactive elements
- Transition easing consistent: `0.3s cubic-bezier(0.4, 0, 0.2, 1)`

### Weaknesses
- Gold text on wine background needs contrast verification
- No dark mode
- Some inline styles override the CSS system (detail pages)

## The One Thing That Matters Most

**Fix #1: Wire the form to actually send data.** Everything else is polish. If the form doesn't work, the business has a beautiful brochure that generates zero leads. This is a 30-minute fix with Netlify Forms (add `data-netlify="true"` to the form element) or a 2-hour fix with a serverless function.

Until that's done, every visitor who fills out the form and sees "We'll reach out within 24 hours" is being lied to.
